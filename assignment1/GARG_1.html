Inline Question 1:¶

It is possible that once in a while a dimension in the gradcheck will not match exactly. What could such a discrepancy be caused by? Is it a reason for concern? What is a simple example in one dimension where a gradient check could fail? Hint: the SVM loss function is not strictly speaking differentiable

Your Answer: Such a scenario is possible when x is such that loss function is not differentiable (corners of loss function) near x. For example, there exists a corner at x = 0 for max(0, x). In this case analytical gradient will not be equal to numerical gradient near x = 0. Numerical gradient is dependent on which direction we are approaching to compute the gradient. It should not be a reason for concern since we are using analytical gradient for gradient descent computations

Inline question 2:¶

Describe what your visualized SVM weights look like, and offer a brief explanation for why they look they way that they do.

Your answer: Weights capture approximate boundaries (or features) of the object to be the classified. Fourth dimension of weight is 10 which is equal to number of classes to be classified. Effectively, each one of these (10) fourth dimension capture discriminating features between classes.

Additionally, this analogy may not be true for all the object classes as it depends upon amount of traning data for a particular class and quality of feature extraction. For instance, if we look at weights of class car, we can observe a figure resembling car. However, this is not the case for weights of class plane.
